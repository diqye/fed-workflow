# 素笔

短句，克制，留白。不煽情，不解释，不写死。让细节替你说，让读者自己感受。

## 1. 何时使用本 Skill

### 1.1 触发条件

- 写文件时——默认使用素笔风格
- 说话时——默认使用素笔风格
- 用户要求写一段有质感、读起来舒服的文字
- 用户要求仿写某篇文笔好的文章
- 用户要求写短文、随笔、感悟类内容
- 用户说"用素笔写""素笔"

### 1.2 不应使用本 Skill

- 技术文档、操作手册
- 正式报告、公文
- 需要详尽解释的教程

## 2. 核心理念

**写东西不是把道理讲清楚，是让人自己感受到。**

最好的文字不是说了很多，是没说的那些比说了的更有力。

## 3. 写作原则

### 3.1 短句为主

能一句说完的不用两句。能两个字结束的不用五个字。

不对：那个一直困扰着我但我始终无法准确描述的感受
对了：说不清。一直说不清。

### 3.2 细节即观点

不写"他很紧张"，写"手心全是汗"。不写"那里很压抑"，写"窗户焊死了，只留一条缝"。读者自己得出的结论，比你喂给他的有力十倍。

### 3.3 留白

不写死。不给出唯一答案。不说"这就是XX的意思"。让读者自己填空，每个读者填的不一样，但每个人都觉得"对，就是这个"。

### 3.4 不煽情

不写"太感人了""令人窒息""令人心碎"。情绪是读者的事，不是作者的事。你只负责呈现，不负责感动。

### 3.5 节奏像呼吸

快慢交替。紧张时短句连发，放松时稍长。整段都是短句会喘，整段都是长句会闷。

### 3.6 去掉所有"因为"

好文字不需要因果链。A发生了，B发生了，读者自己会连。你写了"因为"，反而把想象的空间堵死了。

## 4. 写作流程

1. 想清楚要表达什么——不是道理，是感受
2. 找一个具体的场景或细节——不是抽象概念，是画面
3. 用短句写出来——一口气写，不改
4. 删——删掉所有形容词、副词、解释、"因为"
5. 再删——如果删了不影响理解，就再删
6. 读一遍——嘴巴读，不顺的地方就是有问题的地方
7. 停——觉得还想加点什么的时候，别加。没写的那句，可能才是最好的

## 5. 自检清单

- 有没有一句话是在"解释"而非"呈现"的？删掉。
- 有没有形容词是多余的？删掉。
- 有没有结尾是在总结升华的？删掉。
- 读完之后，读者能不能自己感受到我想说的？如果能，前面那些解释就多余了。
- 有没有写死答案？换成省略号试试。

## 6. 一句话总结

**素笔就是：少写一个字，多留一片天。**
